HEVN CLI is a standalone command-line wrapper for the HEVN backend API and MCP transfer endpoints. It is built for operators, finance teams, developers, and AI agents that need scriptable access to HEVN account data and payment workflows.

## What you can do

- Authenticate with `hevn login` and store a local app API key.
- Inspect account, KYC, balance, card, bank, contact, invoice, and contract state.
- Create contacts for email, crypto, and fiat payout destinations.
- Create invoices manually, from uploaded invoice documents, or from contracts.
- Generate HEVN contracts from templates or upload existing signed contracts.
- Move USDC through MCP by invoice, email recipient, contact, or payout quote.
- Emit human-readable tables or machine-readable JSON/YAML for automation.

## Command shape

Most commands follow this pattern:

```bash
hevn [global-options] <resource> <action> [options]
```

Common global options:

```bash
hevn --env dev account get
hevn --base-url https://api.example.com/api/v1 account get
hevn --debug transfer --invoice-id <invoice-id>
```

Most read and write commands also support:

```bash
--json
--yaml
```

Use these flags for scripts, agent tools, CI jobs, and data extraction.

## Authentication model

`hevn login` opens the HEVN app in a browser, creates or selects an app, issues an app API key, and saves it in your local HEVN CLI config. App API keys use the `X-Api-Key` header for the backend API and `X-API-Key` for MCP transfer endpoints.

Legacy bearer tokens are still supported through `HEVN_API_KEY` with `Authorization`, but app API keys are the preferred path.

<Note>
  MCP transfer commands require an app API key and an app id. Run `hevn login` first unless your automation sets both values explicitly.
</Note>
